# תיעוד סנכרון רציף

תיקייה זו מתעדת את פתרון הסנכרון הרציף מ-Dynamics 365 / Dataverse אל PayPlus.

מנוע הסנכרון מוגדר במכוון כפתרון ממוקד. הוא מיועד לסנכרון יוצא ומבוקר מ-Dynamics 365 אל PayPlus עבור אובייקטים עסקיים שיש להם יעד ברור ב-PayPlus וסיבה תפעולית ברורה להתקיים שם.

## מסמכים

| מסמך | מטרה |
| --- | --- |
| [תפיסת פתרון](solution-concept.md) | תפיסה עסקית, מטרות, גבולות תהליך והסבר איך להבין את הסנכרון עבור צוות עסקי וצוות יישום. |
| [ארכיטקטורה ורכיבים](architecture-and-components.md) | ארכיטקטורה טכנית, זרימת זמן ריצה, טבלאות Dataverse, אחריות הפלאגין, אחריות Power Automate ודיאגרמות. |
| [גבולות סנכרון ו-Governance](sync-scope-and-governance.md) | יעדי סנכרון מומלצים, טבלאות שלא מומלץ לסנכרן גנרית, קריטריוני החלטה וכללי עבודה. |

## תקציר בשורה אחת

Dynamics 365 נשאר מקור האמת. פלאגין Dataverse בודק מיפוי פעיל וכללי סינון, כותב שינויים שעברו תנאים ל-Sync Outbox, ו-Flow גנרי שולח את פריטי ה-Outbox ל-PayPlus דרך מחבר Sandbox או Production.
